# MVP Plan

CodeMong의 초기 MVP는 완성형 서비스를 목표로 하기보다, 학습 진단의 핵심 흐름이 실제로 작동하는지 검증하는 데 초점을 둡니다.

---

## 1. MVP Goal

사용자가 문제를 풀고, 풀이 기록이 저장되며, 오답 기록을 바탕으로 취약 개념이 계산되는 최소 기능을 구현합니다.

핵심 검증 질문은 다음과 같습니다.

1. 사용자의 풀이 기록을 구조화해서 저장할 수 있는가?
2. 오답 기록을 기준으로 취약 개념을 계산할 수 있는가?
3. 진단 결과를 생성형 AI가 이해하기 쉬운 문장으로 설명할 수 있는가?

---

## 2. MVP Scope

| Priority | Feature | Description |
|---|---|---|
| P1 | 문제 목록 조회 | 사용자가 학습 문제를 확인 |
| P1 | 답안 제출 | 사용자의 답안을 제출하고 정오답 판정 |
| P1 | 풀이 기록 저장 | 사용자별 풀이 결과 저장 |
| P1 | 취약 개념 계산 | 오답 기록 기반 취약 단원 분석 |
| P2 | 오답 유형 분류 | 문법 실수, 개념 부족, 로직 오류 구분 |
| P2 | AI 진단 설명 | 취약 개념과 학습 방향을 자연어로 설명 |
| P3 | 학습 대시보드 | 사용자별 학습 현황 시각화 |

---

## 3. Acceptance Criteria

| Feature | Success Criteria |
|---|---|
| 문제 목록 조회 | `/problems` 호출 시 문제 목록 반환 |
| 답안 제출 | `/submit` 호출 시 정답 여부 반환 |
| 취약 개념 계산 | 사용자별 오답 개념이 집계되어 반환 |
| 진단 로직 | 정답 여부, 풀이 시간, 시도 횟수, 오답 유형을 기준으로 취약도 점수 계산 |
| AI 설명 | 진단 결과를 사용자가 이해하기 쉬운 문장으로 변환 |

---

## 4. Development Order

1. DB Schema 확정
2. FastAPI 기본 라우터 구현
3. 문제 조회 및 답안 제출 기능 구현
4. 풀이 기록 저장 구조 연결
5. 취약도 계산 로직 구현
6. 생성형 AI 설명 기능 연동
7. 사용자 대시보드 확장
