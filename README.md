# CodeMong - AI 기반 프로그래밍 학습 진단 서비스
> 본 저장소는 졸업작품 CodeMong의 문제 정의, 데이터 구조, FastAPI 기반 백엔드 흐름을 정리한 초기 설계 및 데모 저장소입니다.  
> 완성형 서비스가 아니라, SKALA 지원 및 향후 구현 확장을 위해 핵심 구조를 정리한 자료입니다.

## 1. 프로젝트 개요

CodeMong은 프로그래밍 초보자가 자신의 취약 개념을 정확히 파악하지 못해 학습 방향을 잃는 문제를 해결하기 위한 학습 진단 서비스입니다.

사용자의 풀이 기록과 오답 유형을 바탕으로 어떤 단원과 개념에서 어려움을 겪는지 진단하고, 다음 학습 방향을 제안하는 것을 목표로 합니다.

## 2. 문제 정의

프로그래밍 초보자는 문제를 틀렸을 때 그 원인이 단순 문법 실수인지, 개념 이해 부족인지, 문제 유형에 익숙하지 않은 것인지 구분하기 어렵습니다.

이로 인해 많은 문제를 풀어도 자신의 약점을 명확히 알지 못하고, 반복적으로 비슷한 오류를 경험하게 됩니다.

## 3. 핵심 기능

- 문제은행 기반 학습 문제 제공
- 사용자의 풀이 기록 저장
- 오답 유형, 소요 시간, 시도 횟수 기반 학습 데이터 누적
- 단원별 취약도 진단
- 생성형 AI를 활용한 진단 결과 설명
- 단계별 힌트 및 학습 방향 추천

## 4. AI 활용 방향

CodeMong에서 AI는 정답을 대신 제공하는 역할이 아니라, 사용자의 학습 데이터를 바탕으로 진단 결과를 이해하기 쉽게 설명하고 다음 학습 행동을 제안하는 보조 역할을 담당합니다.

AI가 모든 판단을 대신하는 구조보다, 풀이 기록과 오답 유형 같은 데이터 기반 판단을 우선하고 AI는 설명과 추천을 보조하는 방식이 더 신뢰성 있다고 판단했습니다.

## 5. 데이터 구조

주요 데이터는 다음과 같이 구성합니다.

- User: 사용자 정보
- Problem: 문제 정보
- Concept: 단원 및 개념 정보
- Submission: 풀이 기록
- MistakeType: 오답 유형
- Diagnosis: 취약 단원 진단 결과

## 9. API Demo Structure

현재 FastAPI 데모는 CodeMong의 핵심 학습 흐름을 단순화해 구성했습니다.

| Method | Endpoint | Purpose |
|   GET  | `/`      | API 상태 확인 |
|   GET  | `/problems` | 문제 목록 조회 |
|   POST | `/submit` | 사용자 답안 제출 및 정오답 판정 |
|   GET  | `/diagnosis/{user_id}` | 사용자별 취약 개념 진단 결과 조회 |

이 구조는 문제 풀이 -> 풀이 기록 저장 -> 취약 개념 분석 -> AI 기반 설명/추천으로 이어지는 CodeMong의 핵심 흐름을 표현합니다.

## 10. Diagnosis Logic

CodeMong의 진단은 단순 정답률만 보는 것이 아니라, 다음 데이터를 함께 고려하는 방향으로 설계했습니다.

- 정답 여부
- 오답 유형
- 풀이 소요 시간
- 시도 횟수
- 단원 및 개념 정보

예를 들어 같은 오답이라도 단순 문법 실수인지, 개념 이해 부족인지, 문제 유형 미숙인지에 따라 제공해야 할 힌트와 학습 방향이 달라집니다.  
따라서 AI가 모든 판단을 대신하기보다, 풀이 기록 데이터를 기반으로 취약 개념을 우선 판단하고 AI는 그 결과를 설명하는 보조 역할을 맡도록 설계했습니다.
